# AI星月上游内容迁移需求

## 目标

把 AI星月 APK 当前依赖 `https://aifun.wiki/` 的内容接口迁移到 `https://patcher.villainy.top/` 后端数据库中，使内容可由自有后端控制、修改，并逐步做到不依赖上游服务器。

## 范围

- 将已知内容接口的 JSON 响应缓存到后端 SQLite。
- 后端读取内容时优先从本地数据库返回，并继续执行 AI星月品牌替换。
- 提供同步/估算脚本，统计 JSON 体量、接口条数、媒体 URL 数量和可探测媒体大小。
- 支持 `cache_first` 与 `local_only` 两种模式：
  - `cache_first`：本地有缓存就返回本地，无缓存时回源并写入缓存。
  - `local_only`：只返回本地缓存或 fallback，用于验证完全脱离上游。

## 验收标准

- 后端存在内容缓存表，且不会破坏现有用户、积分、充值数据。
- 同步脚本可以对探索页、帖子、公告、活动、站点、表情、模型等接口做抓取/入库/估算。
- 可在服务器上生成存储估算报告，至少拆分 JSON 数据和媒体资源两部分。
- `patcher.villainy.top` 接口能从本地库返回已同步内容。
- 切换到 `CONTENT_MODE=local_only` 后，已同步的核心页面接口不再访问 `aifun.wiki`。

## 非目标

- 不直接修改 APK 图标、welcome 页和已完成的 AI星月资源。
- 不在本阶段搭建完整可视化内容管理后台。
- 不盲目下载全部媒体文件；先估算空间，再按结果决定是否镜像。
