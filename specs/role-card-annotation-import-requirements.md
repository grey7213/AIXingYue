# AI星月角色卡标注回填与展示 Requirements

Updated: 2026-07-11

## 目标

- 接收人工整理后的 8778 张公开官方角色卡 ZIP，按稳定 `local_apps.id` 安全回填标准化标签。
- 读取文件名末尾三位 `√/✗`，顺序固定为：开场白、世界书、正则。
- 在探索页和角色详情页清晰展示三项能力，同时提升暖纸橙金主题的排版、层次、配色和交互质感。

## 数据规则

- 标签的唯一标准来源是 ZIP 内 `tag-overrides.filled.csv` 的 `new_tags_json`；不能仅从文件名反推，因为 `NTR/NTL`、`人妻/熟女` 等组合标签在文件名中会拆分。
- `action=replace` 时完整替换 `local_apps.tags`；`action=skip` 时保持原标签。
- 文件名三位标记只写入独立角色标注，不混入分类标签：
  - 第一位：主开场白 `opening_statement`。
  - 第二位：`extra_settings.world_info`。
  - 第三位：`extra_settings.regex_scripts`。
- 回填前必须校验 Manifest SHA、卡 JSON SHA、内部 ID、展示 ID、原标签和导出时 `updated_at`。
- 本次不修改角色正文、开场白、世界书、正则、封面、来源、公开状态、排序时间或所有者。

## 删除边界

- 本次不删除任何角色卡。
- `√✗✗` 的 1006 张卡仅作为“手枪卡/仅主开场”宽候选，不等于空卡。
- 后续删除必须单独出清单并经人工确认。

## UI 验收

- 探索页推荐卡与普通卡显示紧凑、可读的“开场 / 世界书 / 正则”能力状态，不能只靠颜色表达。
- 角色详情页显示完整能力面板和分类标签。
- 卡片层级、留白、封面遮罩、边框、阴影、hover/focus、空态与深色主题统一。
- 375、768、1024、1440 宽度无横向溢出；支持 `prefers-reduced-motion`。

## 非目标

- 不改 APK。
- 不重新解释或改写角色内容。
- 不恢复用户 BYOK/API Key 输入。
