# 惑梦社区工坊、不可变版本与 Spine 稳定化需求（2026-07-20）

## 目标

- 合并新交接 ZIP 中的历史会话三点菜单、社区工坊、对话 Mod、作品版本和 Spine 动画能力。
- 社区作品支持 `mod`、`ui_template`、`preset`；角色卡继续使用 `local_apps`，但与社区作品共享不可变版本语义。
- 所有新会话固定到开始时选择的角色版本；进行中不得跟随作者后续更新。
- 对话 Mod 固定到所选 Mod 版本，世界书顺序恒为：平台反扒卡 → 用户排序的 Mod → 角色卡自身世界书。
- Spine 资源包可校验、托管、切换和回退，连续切换、窗口缩放及临时加载失败不导致运行时永久失效。

## 功能验收

### 历史会话

- 历史行不再展示点赞和独立“继续”按钮；点击会话主体仍可继续。
- 收藏、复制、删除进入无边框灰色半透明三点菜单；菜单是锚定按钮的小弹层，不占满屏。
- 同时只展开一个菜单；点击外部、进入会话或完成操作后关闭。

### 社区工坊

- “新建”弹窗可进入角色、Mod、UI 模板、预设创建。
- 公共社区可按类型查看作品；创作者可管理自己的作品；用户可收藏 Mod/UI 模板/预设。
- Mod 发布页只公开名称、简介和条目名；正文只对作者、管理员或开源作品可见。
- UI 模板公开安全演示；演示不得获得主站同源、账号、storage 或网络能力。
- 预设详情公开条目名称，不公开正文；开源后正文只读展示。
- 角色卡可标记开源、报名当前赛事，并选择一个已收藏预设和多个已收藏 UI 模板。
- 管理员可发布带标题、内容、奖励、起止时间的赛事；赛事榜、角色详情均显示票数并支持有效期内投票。

### 版本

- 新建角色或社区作品时自动产生不可变 `v1`。
- 后续发布必须输入版本名和作者介绍；发布事务直接从提交内容生成新快照，不能先覆盖当前版本。
- 角色/作品详情展示历史版本名、作者介绍、发布时间；可查看有权访问的版本详情。
- 新会话即使选择“最新版”，服务端也解析为当时的具体版本 ID 并永久写入会话。
- 版本 ID 必须属于当前角色/作品；跨作品版本不可使用。
- 复制会话继承角色版本和 Mod 版本；进行中不提供更换版本接口。
- 历史版本引用的媒体不可因当前版本删除素材而物理失效。

### Spine

- `.spine.zip` 必须包含一个 skeleton、一个 atlas 和 atlas 实际引用的全部纹理。
- skeleton 导出版本与站点本地 runtime 主次版本兼容；不接受未知版本的二进制骨骼。
- 前端仅加载本地固定 runtime，不使用 CDN 通配版本。
- 连续切换两个 Spine 立绘、快速切换、ResizeObserver、页面隐藏/恢复、WebGL context lost 后可恢复或安全降级。
- 单个素材失败只降级该素材，不永久禁用整个 Spine 图层。

## 安全与非目标

- 保持 `CONTENT_MODE=local_only`、普通用户 BYOK 关闭、模型 API Key 不出服务端。
- UI 模板演示与卡片高级渲染均不得增加 `allow-same-origin`，不得开放外部网络。
- 不部署 ZIP 内 SQLite、WAL/SHM、日志、pyc、测试账户或离线开发数据。
- 本轮不修改 APK、历史技术服务名或数据库文件名。
