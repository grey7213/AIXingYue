# AI星月 Web 全站 UI 设计交接

## 设计来源

- Manual prototype：`output/ui-reference-character-editor/创作（角色卡编辑器）/index.html`
- UI/UX 辅助规范：`design-system/ai星月-web/MASTER.md`
- 视觉优先级：用户提供原型 > AI星月现有品牌资产 > 通用推荐库。

## 视觉方向

- 气质：轻盈、温暖、精致、工具感明确，不做高饱和游戏化界面。
- 画布：中性浅灰 `#F5F5F7`，内容卡片纯白。
- 主色：AI星月玫红 `#FF2E63`；悬停/渐变辅助 `#FF6B8B`。
- 文字：主文字 `rgba(0,0,0,.85)`，辅助 `.55`，弱提示 `.38`。
- 边框：`#E6E6EA`；分隔 `#F0F0F2`。
- 圆角：输入与按钮 8px，卡片 12px，大抽屉/模态 16px。
- 阴影：`0 2px 12px rgba(0,0,0,.06)`，避免厚重浮层。
- 字体：系统中文字体栈；代码/JSON 使用等宽字体，不全站使用 Fira Code。

## 比例与布局

- 桌面应用内容最大宽度 1180–1280px；编辑器正文阅读宽度 900px。
- 桌面侧栏 220px；顶部固定工具栏 64px；聊天正文最大宽度 900px。
- 页面基础间距 20px；卡片内边距 18–20px；组件间距以 4/8/12/16/20/24 为阶梯。
- 移动端左右边距 14–16px，固定操作区必须避让安全区。

## 页面策略

- 首页/探索：减少大色块，角色卡采用白底、细边框、克制封面比例和玫红状态标签。
- 聊天：顶部固定快捷操作；消息区弱化装饰，输入区保持悬浮白卡质感。
- 创作工具：最接近参考原型，使用吸附工具栏、分段标签、白色字段卡片、右侧预览抽屉。
- 我的/历史/收藏/信息中心：统一列表密度、空状态和筛选控件。
- 管理后台：沿用同一令牌，但数据表与图表保持更高信息密度。

## 交互状态

- Hover 只改变颜色、边框或阴影，不使用导致布局位移的缩放。
- Focus-visible 使用 2px 玫红外环。
- Loading 使用稳定骨架屏，不闪烁代码或未初始化模板。
- 动画 150–250ms，并支持 `prefers-reduced-motion`。

## 禁止项

- 大面积粉色背景、过多渐变、玻璃透明度过低。
- Emoji 作为功能图标；混用多套图标。
- 过圆胶囊化所有控件、过厚阴影、卡片层层套卡片。
- 为追求视觉而隐藏关键字段或改变现有操作流程。

## 视觉 QA

- 首页、探索、创作、聊天、我的、信息中心、后台各至少一张桌面截图。
- 首页、创作、聊天、我的各至少一张 375px 移动截图。
- 检查固定栏遮挡、输入框高度、长中文、空状态、弹窗和抽屉。
