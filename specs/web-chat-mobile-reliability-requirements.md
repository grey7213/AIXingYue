# AI星月 Web 聊天可靠性与沉浸式首页需求

Updated: 2026-07-10

## 目标

- 仅修改 Web App，不修改 APK，也不恢复已下架的 APK 下载渠道。
- 修复“AI续写”报错和复读上一条回复的问题。
- 修复流式连接被截断、页面切后台/回前台后显示旧回复或串会话的问题。
- 缩短验证码接口等待时间，并保持真实投递失败可追踪。
- 探索页角色卡每轮随机稳定分页，主动“换一批”时立即展示新一轮结果，旧请求不得覆盖新请求。
- 登录后的 `/app/` 默认进入上次打开的对话；无历史对话时进入探索页。
- 聊天页隐藏通用底部导航，使用固定顶部栏：左侧功能菜单、中央角色/会话标题、右侧模型/记忆/渲染菜单。

## 用户流程

### AI 续写

- 末条 assistant 消息提供独立“AI续写”动作。
- 续写必须保留上一条 assistant 内容作为上下文，并追加“从上一条结尾自然继续、不要复述”的内部指令。
- 续写生成一条新的 assistant 消息，不覆盖原消息，不创建可见的“继续”用户气泡。
- 只有开场白、还没有 user 消息时也能续写。

### 流式可靠性

- 前端只接受当前会话、当前生成序号的 SSE 事件。
- SSE 未收到 `message_end` 就断开时视为截流，不得把旧回复当成本次成功结果。
- `message_end.reply` 是最终权威内容，应覆盖已拼接的临时 delta。
- 生成期间切后台/回前台不得整体替换当前消息数组；结束后从服务端对账。

### 首页与顶部菜单

- `/app/` 登录态：优先进入 localStorage 记录的上次打开会话；记录失效时进入服务端最近会话；无会话时进入 `/app/explore.html`。
- `/app/explore.html` 承载原探索首页内容，不能再重定向回 `/app/`。
- 聊天页不显示桌面全局侧栏和移动底部导航。
- 左上菜单至少包含：探索、我的、历史对话、收藏、当前会话列表。
- 右上菜单至少包含：更换模型、长期记忆、高级渲染、当前角色收藏。

### 验证码

- HTTP 发码接口不能被 SMTP 握手长期阻塞。
- 优先使用 Resend HTTPS API（复用服务端现有凭据，密钥不回显）；未配置时保留 SMTP fallback。
- 邮件发送结果写日志；投递失败时删除本次验证码，避免用户拿到一个无法接收的有效码。

### 探索随机

- 同一轮 page 1/page 2 使用相同 seed，不重复。
- 筛选或“换一批”会生成新 seed，并让之前未完成的请求失效。
- 从角色详情返回时可以恢复原列表和滚动位置；主动换一批时不得恢复旧列表。

## 验收标准

- 本地假上游验证续写请求包含上一条 assistant，且保存为新 assistant 消息。
- 模拟 SSE 截流时前端显示明确错误并完成服务端对账；模拟 `delta + message_end(full)` 时最终气泡等于 full。
- 430x932 浏览器截图中顶部栏固定、左右菜单可用、无底部导航、输入框不被遮挡。
- `/app/` 有历史时进入上次会话，无历史时进入探索。
- 连续快速切换探索筛选时只有最后一个请求结果生效；两次“换一批”首屏 ID 集合不同。
- 发码接口响应耗时显著低于旧 SMTP 同步链路，并验证服务/Nginx/公网 health 正常。

## 非目标

- 不修改、不构建、不发布 APK。
- 不恢复 `/download/` 或充值兑换渠道。
- 不开放普通用户自定义 API Key。
- 不降低 Tavo sandbox 的 CSP、无 same-origin 隔离和脚本确认边界。
