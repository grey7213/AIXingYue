# 惑梦农场替换签到需求

## 目标

- 使用用户提供的 `E:\obs录制\农场.zip` 视觉与交互方向，将 Web 端“每日签到”替换为可持续游玩的惑梦农场。
- 农场与现有惑梦账号、惑梦币余额和北京时间自然日规则打通，不要求再次绑定用户 ID 或 API Key。
- 保持旧 APK 签到接口兼容，不重打 APK，不影响历史签到记录、聊天记录、充值记录和用户余额。

## 用户与范围

- 已登录用户可查看农场、种植、浇水、收获、查看系统好友农场并进行有限次数的采摘互动。
- 农场币、经验、体力、作物成熟时间、连续活跃天数和每日奖励全部由服务端计算并持久化。
- 农场币只用于农场内部种子和成长，不等同于惑梦币。
- 每个北京时间自然日第一次有效收获可领取后台配置的 `daily_points` 惑梦币；同日其他收获只给农场币和经验。
- 第一版好友为系统 NPC，不读取或转移真实用户资产。

## 兼容与安全约束

- 保留 `daily_reward_claims(user_id, claim_date)` 作为每日惑梦币唯一账本。
- 旧 `/console/api/ctf/dailyapppoints`、`points-claim` 和 `/console/api/web/rewards/daily` 继续可用，并与农场每日收获共享同一唯一约束。
- 历史签到记录视为对应日期已领取农场每日惑梦币，不能删除或重置。
- 种植、浇水、收获、NPC 采摘均要求服务端鉴权；浏览器不能提交可信余额、奖励、成熟时间或用户 ID。
- 所有动作支持 `Idempotency-Key`，同一个键重复请求返回同一结果，不重复扣体力、农场币或发奖励。
- 不引入原型中的 API Key 输入、本地账号绑定、本地余额或 localStorage 权威状态。

## 验收标准

- 新增 `/app/farm.html`，桌面和 390px 移动端均可正常使用，无横向溢出和浏览器错误。
- 新用户首次进入获得基础农场状态、1 块土地和可完成的首轮种植；连续活跃逐步解锁最多 8 块土地。
- 作物成熟、浇水加速、体力恢复、收获收益和每日次数均由服务端时间决定。
- 同一动作并发/重试不重复结算；同一用户同一自然日最多增加一次 `daily_points` 惑梦币。
- Rewards、我的页、Dashboard 和 App 导航中的签到入口改为农场入口；充值、爱发电、在线支付宝、兑换码和订单查询保留。
- 后台 `daily_points` 继续可配置，文案明确为“每日首次有效收获奖励”，仍影响旧 APK 签到适配接口。
- 本地编译、API 测试、旧接口兼容测试、浏览器桌面/移动端截图、线上服务与健康检查全部通过。

## 非目标

- 不在本次实现真实好友关系、玩家间资产转移、农场币兑换惑梦币、排行榜派奖或 APK 内原生农场页。
- 不迁移原型中的假 API 调用统计、开发者中心或虚假的实时天气数据。
