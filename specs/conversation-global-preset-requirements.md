# 单会话全局预设开关需求

## 目标

在 Web 聊天设置面板增加“关闭全局预设”操作。关闭后，仅当前普通对话不应用后台当前启用的全局 Prompt Preset 和全局 Regex Preset。

## 范围

- 开关按普通单人会话持久化，刷新、重新登录、切换会话后保持。
- 历史会话和新会话默认继续应用全局预设，用户主动关闭后才停用。
- 仅影响当前会话；其他会话、其他用户和群聊不受影响。
- 普通发送、流式发送、AI 续写、重新生成和生成新 swipe 使用同一会话设置。
- 关闭时不应用全局 Prompt 的 system_before/system_after/post_history，也不执行全局 Regex 的输入、历史和输出处理。
- 角色卡自身的系统提示词、Prompt Manager、世界书、角色 Regex、Tavo 渲染、长期记忆和站点必需反扒世界书继续生效。

## 验收标准

- 设置面板显示当前会话状态；关闭后按钮明确显示已关闭，重新开启后恢复。
- 会话 A 关闭后不注入全局 Prompt/Regex；会话 B 及群聊仍按后台全局预设运行。
- A 刷新、切换回来或重新登录后仍保持关闭；复制 A 时副本继承状态；新会话默认开启。
- Galgame 选项与本开关独立保存。全局预设关闭时 Galgame 条目不生效，重新开启后恢复该会话原 Galgame 状态。
- 开关不修改后台全局预设 JSON、active_id、更新时间、角色卡数据、消息、积分或会话排序时间。
- 非会话所有者不能读取或修改该设置。
- 390px 与桌面宽度下设置面板无横向溢出，切换过程无浏览器控制台错误。

## 非目标

- 不编辑、删除或切换后台全局预设。
- 不关闭角色卡自身能力或站点必需世界书。
- 不把该开关扩展到群聊。
