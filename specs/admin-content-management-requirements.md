# AI星月后台内容管理需求

## 目标

- 管理端可以上传/导入角色卡，不再只能手动逐项填写。
- 管理端可以可视化管理平台主要内容，至少覆盖当前核心资产：官方角色、用户角色、同步角色、模型配置、用户、积分/兑换码、请求日志。
- 角色内容管理要支持搜索、来源筛选、状态筛选、预览、编辑、上下架、公开/私有、删除。
- 保持当前 AI星月后台 UI 风格，不重做品牌、配色或导航。

## 角色卡上传范围

- 支持上传 `.json` 文件。
- JSON 可以是单个对象、对象数组，或包含 `items` / `list` / `cards` / `apps` 的对象。
- 每张角色卡至少需要 `name`；其它字段尽量兼容：
  - `summary`
  - `description`
  - `opening_statement`
  - `pre_prompt`
  - `tags`
  - `cover_url` / `cover` / `icon`
  - `suggested_questions`
  - `llm_model`
  - `sort_weight`
  - `status`
  - `is_public`

## 管理能力

- 管理员可以创建官方角色卡。
- 管理员可以导入多张官方角色卡。
- 管理员可以编辑任意来源的本地角色卡，包括同步角色、用户角色、官方角色。
- 管理员可以勾选多张本地角色卡后批量修改标签、简介、角色设定和世界书内容。
- 批量标签修改支持替换、追加和移除；批量简介/角色设定必须显式勾选后才覆盖。
- 批量世界书修改支持整体替换、追加条目、按 `id/name/title/comment` 合并条目，并复用现有世界书规范化规则。
- 管理员可以删除任意来源的本地角色卡。
- 管理员可以快速切换角色公开状态、发布/草稿状态。
- 管理员操作不自动触碰 APK 文件；角色数据来自服务器 SQLite。

## 验收标准

- 后端提供导入 API，并返回成功条数和错误列表。
- 后端管理员更新/删除 API 可作用于 `admin`、`user`、`upstream` 三类 `local_apps`。
- 后端管理员批量更新 API 只作用于请求中指定的角色 ID，返回 requested、updated、not_found、errors 等统计。
- 前端后台角色卡 Tab 有上传 JSON 入口、导入预览/结果、可视化列表和快捷管理按钮。
- 前端后台角色卡 Tab 支持当前页勾选、全选、清空选择和批量编辑弹窗。
- 本地和线上验证：导入测试角色、编辑、上下架、删除均成功。
- 本地和线上验证：临时角色批量标签追加/移除/替换、简介/角色设定覆盖、世界书追加/合并/替换均成功，未选中角色不受影响。
- 部署后 `/admin.html` 正常打开，浏览器无关键 JS 错误。

## 非目标

- 不做复杂 CMS 工作流审批。
- 不做真实支付后台对账。
- 不自动同步或覆盖 APK 内置资源。
