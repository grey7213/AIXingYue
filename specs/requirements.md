# 安卓逆向工程学习项目 - 需求文档

## 当前 CTF 授权验证目标（2026-06-14）

目标是在已授权的 CTF 沙箱资产范围内，验证目标 APK 的控制权限、完整性校验/反篡改边界，以及本地签名打包链路是否能够重放到生产式流程。

### 本次范围

- 检查本地工作区是否能任意修改 APK 解包产物、核心 smali、manifest、资源、加壳 stub 和签名产物。
- 检查是否存在已连接 Android 设备/模拟器，以及是否具备 root/管理员级运行时控制。
- 对 `base.apk` 执行最小篡改、重打包、zipalign、临时证书签名、签名验证。
- 如 adb 设备存在，继续执行安装、启动、logcat 抓取、root 检查和加壳运行时失败点验证。
- 输出可重复执行脚本和结果报告，用于后续检测策略设计。

### 本次验收标准

- 明确区分“本地文件控制权限”和“目标运行环境 root/管理员权限”。
- 产出可执行 PowerShell 脚本，能自动完成工具发现、签名检查、篡改构建、重签名和可选 adb 验证。
- 产出审计报告，说明 `StubApp`、`libjiagu*.so`、`.jgapp`、APK v2/v3 签名、Flutter `libapp.so` 的安全边界。
- 若无法做动态验证，必须给出真实阻塞原因和一键复跑命令。

## 项目目标

学习安卓逆向工程技术，通过分析真实APK应用的内部功能和内核实现，掌握：
- APK文件结构和打包机制
- Dalvik/ART字节码分析
- 安卓应用架构模式识别
- 代码混淆与反混淆技术
- 原生库（Native Library）逆向
- 安全机制与保护手段识别

## 目标用户

- 安卓开发者希望深入理解系统底层机制
- 安全研究人员学习应用安全分析
- 软件工程师研究架构设计模式

## 分析范围

### 包含内容
1. **APK结构分析**
   - AndroidManifest.xml 解析
   - 资源文件结构（res/）
   - DEX文件分析
   - 原生库文件（.so）
   - 签名和证书信息

2. **代码逆向**
   - Java/Kotlin 源码反编译
   - 核心业务逻辑识别
   - 架构模式分析（MVVM/MVC/Clean Architecture）
   - 关键算法提取

3. **内核功能分析**
   - 应用启动流程
   - 核心服务和组件
   - 网络通信机制
   - 数据存储方案
   - 安全防护措施

4. **技术学习总结**
   - 可借鉴的架构设计
   - 代码组织方式
   - 性能优化技巧
   - 安全最佳实践

### 不包含内容
- 破解或绕过应用保护机制用于非法目的
- 提取或泄露敏感用户数据
- 重新打包并分发应用
- 商业逆向竞争分析

## 验收标准

1. ✅ 成功解包两个APK文件并提取所有资源
2. ✅ 反编译得到可读的Java/Kotlin源代码
3. ✅ 生成完整的代码结构分析报告
4. ✅ 识别出核心业务逻辑和架构模式
5. ✅ 提取关键技术实现细节
6. ✅ 总结可学习的技术要点和最佳实践
7. ✅ 创建学习笔记和知识库

## 工具清单

- **apktool** - APK解包和资源提取
- **jadx / jadx-gui** - DEX到Java反编译
- **dex2jar + jd-gui** - 备用反编译方案
- **AXMLPrinter2** - AndroidManifest解析
- **apksigner** - 签名验证
- **Ghidra / IDA** - 原生库逆向（可选）
