# AI星月 Web 角色聊天需求

## 目标

- 让 `https://patcher.villainy.top/` 的网页端成为 AI星月 APK 的同库 Web 客户端。
- APK 注册的邮箱/密码可直接登录网页；网页登录创建的账号也可被 APK API 使用。
- 用户忘记密码时可通过注册邮箱验证码重置密码，重置成功后可直接登录网页。
- 网页端提供移动优先的角色探索、角色详情、聊天、创建角色、管理我的角色和个人积分页面。
- 用户创建的角色使用站点管理员配置的 OpenAI-compatible 模型 API；不要求普通用户填写密钥。

## 用户

- 普通用户：用邮箱密码登录，浏览角色、发起聊天、创建自己的角色卡、管理公开状态和设定。
- 管理员：在后台查看用户/充值/请求日志，并配置站点级模型 API。

## 范围

- 保持现有网页登录/注册页视觉与流程不变。
- 前端范围限定在 `frontend/app/` 和共享 `frontend/assets/js/api.js`。
- 后端范围限定在 `tools/ai_fengyue_local_server.py` 和部署脚本必要配置。
- 站点级模型配置包括：base URL、模型名、API key、温度、是否启用。
- API key 读取时必须脱敏；不在日志、文档、最终回复中回显。

## 验收标准

- `/app/explore.html` 移动端可看到底部导航、搜索、分类、角色卡，点击进入角色详情或聊天。
- `/app/character.html?id=...` 显示封面、简介、标签、开场白，并能进入聊天。
- `/app/chat.html?app_id=...` 能创建/继续会话并保存消息。
- 单次成功角色回复消耗 50 积分；余额不足 50 时不创建新回复并返回明确错误。
- `/app/create.html` 可以创建公开/私有角色卡，留空模型配置时使用站点默认 API。
- `/app/my-apps.html` 可以编辑、删除自己的角色卡。
- `/app/login.html` 支持发送密码重置验证码、校验验证码、设置新密码，并拒绝错误验证码或过短密码。
- `/app/` 高级搜索筛选区在桌面、中等宽度和移动端均不横向溢出，关键词输入可用。
- `/admin.html` 或后台 API 可以读取/保存站点模型 API 配置，key 不明文回显。
- 线上 `health`、探索、角色详情、聊天、创建角色、模型配置接口通过真实请求验证。

## 非目标

- 不重写 APK。
- 不实现复杂推荐算法、实时流式打字、多端 WebSocket 同步。
- 不把用户个人 API key 存入数据库；当前采用管理员站点级配置。
