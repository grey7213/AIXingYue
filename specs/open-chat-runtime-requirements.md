# 惑梦开放式聊天扩展运行时需求

## 目标

把当前“逐条消息 Tavo iframe”升级为可被角色卡或已安装 UI 扩展接管整个聊天区域的开放式运行时。角色卡可以自带字体、背景、气泡、消息列表、输入框、按钮和交互动效；平台继续作为账号、模型、积分、消息和会话状态的权威控制器。

## 用户与场景

- 普通用户：打开支持开放式界面的角色卡后，直接使用角色作者设计的完整聊天 UI。
- 角色作者：通过角色卡 `card_experience.chat_shell` 或社区 UI 模板发布完整聊天界面。
- 管理员：后续可安装经过确认的酒馆助手/Tavo/其他扩展包，由统一运行时提供兼容 API，不再逐个修改聊天页。

## 功能范围

- 支持 `default / legacy-tavo / open-shell` 三种渲染路径，旧卡默认行为不变。
- Open Shell 使用单会话持久化全屏运行时，而不是每条消息创建一个 iframe。
- 卡片可提供 HTML、CSS、JavaScript 和界面元数据。
- 运行时接收当前角色、会话、消息、swipe、生成状态和能力列表的状态快照/增量。
- 运行时可以请求：发送、AI 续写、重新生成、swipe、编辑、删除、回溯、加载更早消息、朗读和打开平台设置。
- 平台继续执行鉴权、扣费、SSE、消息完整性校验、所有权校验和数据库持久化。
- 必须保留一个不受卡片控制的“退出卡片界面”入口；运行时异常时自动回退默认 UI。
- 后续扩展包使用同一 capability API；兼容层可以增加 SillyTavern/酒馆助手函数别名。

## 安全与兼容边界

- 开放式是视觉和扩展 API 开放，不等于角色脚本直接进入主页面同源 DOM。
- 卡片运行时不得直接获得账号 Cookie、localStorage、主页面 DOM、管理 API 或模型密钥。
- 父子通信必须校验 `event.source`、随机 channel token、消息 schema、长度和当前会话/消息归属。
- 删除、回溯等破坏性操作由父页面确认。
- 默认卡片运行时禁止任意网络；未来管理员安装的可信扩展按 manifest 权限单独授权。
- 旧 Tavo/Regex/Prompt Template、续写、回溯、重生成和 swipe 必须回归通过。

## 验收标准

- 一个测试角色卡可以用自带 UI 完整渲染消息、输入框、字体、气泡和操作按钮。
- 发送、续写、重生成、swipe、回溯和加载更早消息通过兼容桥调用现有业务逻辑。
- 桌面与 390×844 移动端无横向溢出；流式消息可增量更新。
- iframe 不含 `allow-same-origin`，卡片脚本不能读取父 DOM/localStorage，不能直接访问平台 API。
- Open Shell 退出或崩溃后默认 UI 可继续工作，历史消息和积分状态不丢失。

## 非目标

- 本阶段不把第三方 SillyTavern 扩展源码直接放进主页面执行。
- 本阶段不开放普通用户 API Key/BYOK。
- 本阶段不重写聊天后端、积分账本或消息表。
