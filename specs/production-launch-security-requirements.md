# 惑梦正式上线注册与安全要求

## 目标

- 解除 250 人内测注册总量上限，允许公开注册。
- 新注册用户赠送 `500` 惑梦币，对应当前 `50` 惑梦币/次约 `10` 次角色回复。
- 保留并加强邮箱、IP、登录、验证码和生成并发防刷。
- 修复正式上线阻断级鉴权、模型密钥泄漏和匿名默认用户回退问题。
- 对 Web、后端、支付、后台和服务器配置形成可复查的上线安全报告。

## 必须满足

- `BETA_MAX_REGISTERED_USERS=0` 表示不限制注册总人数；后台统计明确显示开放注册。
- `NEW_USER_INITIAL_POINTS=500`，只影响新注册用户，不扣减已有账号余额。
- 登录 token 必须服务端签名、带签发时间和过期时间；旧无签名 token 必须失效。
- 未携带有效 token 的私有接口不得回退到默认用户。
- 普通用户角色卡不得控制携带站点 API Key 的上游 Base URL；站点密钥只能发送到管理员预设的目标。
- 管理员、支付订单、角色卡、对话、余额、兑换码等资源继续按当前用户和角色权限隔离。
- 正式域名限制 CORS；请求体、上传和图片代理必须有大小/重定向边界。
- ZPAY 异步通知的验签、金额校验、幂等和原子入账不得削弱。

## 验收

- 本地函数/API 测试证明：旧 token 被拒、新 token 可用、伪造/篡改 token 被拒、过期 token 被拒。
- 无 token 访问私有账户/余额/兑换/聊天接口返回 401，不读写默认用户。
- 用户提交 `api_base_url` 后不会保存或用于站点模型请求。
- 新注册账号初始 `500`，注册人数上限为开放状态，现有邮箱/IP 防刷仍生效。
- 线上服务、Nginx、数据库和支付元数据健康；`CONTENT_MODE=local_only`。
- 桌面/移动端注册文案更新为 `500` 惑梦币、约 `10` 次回复。

## 非目标

- 不重建 APK。
- 不执行真实支付。
- 不自动扣减已有用户额度。
- 不在本轮更换模型供应商或开放用户 BYOK。
