# AI星月运营后台增强需求

## 目标

- 管理员可以在后台给指定用户开放或撤销管理员权限，不需要改服务器环境变量或代码。
- 管理员可以在数据总览里直观看到平台运行情况，而不是只看数字卡片。
- 后台保持当前 AI星月管理台风格，优先增强运营可用性，不重做品牌和导航。

## 范围

- 用户管理：
  - 用户列表显示是否管理员、权限来源、积分余额和注册时间。
  - 管理员可以搜索用户后点击按钮授予/撤销后台权限。
  - 环境变量 `ADMIN_EMAILS` 仍作为兜底超级管理员来源，避免误操作锁死。
- 数据总览：
  - 保留现有核心指标卡。
  - 增加 7 日新增用户、7 日请求量、积分余额构成、角色来源构成、请求状态分布等可视化图表。
  - 图表使用现有静态前端能力实现，不引入重型图表库。
- 后端：
  - `users` 表补充持久化管理员标记。
  - `/admin/api/users` 返回权限状态。
  - 新增 `/admin/api/users/{id}/admin` 用于授权/撤销。
  - `/admin/api/stats` 返回图表所需结构化数据。

## 验收标准

- 本地后端编译通过。
- 管理员 API 可以把普通测试用户设为管理员，再撤销，`/admin/api/users` 能反映状态。
- `ADMIN_EMAILS` 中的用户即使数据库标记为非管理员，也仍可访问后台。
- `/admin/api/stats` 返回 `charts`，包含时间序列和分布数据。
- `/admin.html` 用户管理页有管理员授权/撤销入口，数据总览页有可视化条形图/分布条。

## 非目标

- 本轮不做完整 CMS 审批流。
- 本轮不做站点部署开关、Nginx/服务重启等高危操作的可视化面板。
- 本轮不改变 APK 或前台 App Shell 的品牌与主流程。
